Linux 指令

Linux 指令

基本格式

命令提示符

  • root 位置表示当前用户名
  • ~ 表示根目录，超级用户的家目录：/root，普通用户的家目录：/home/用户名
  • ./ 和 . 开头表示当前工作目录，对于shell脚本不同，一般用./开启新脚本环境
  • ../ 开头表示上一级目录
  • / 开头表示绝对路径
  • #为超级用户提示符，$为普通用户提示符

命令格式

  • Linux 的选项又分为短格式选项（-l）和长格式选项（–all）,短格式前缀一个 - ，可多个连用如-al， 长格式前缀两个-,一次只能指定一个
  • 命令后加 –help 显示用法

显示状况

pwd 显示路径

  • 无参数，打印当前路径

ls 显示目录

  • ls -a 列出文件下所有的文件，包括以“.“开头的隐藏文件

  • ls -l 列出文件的详细信息，如创建者，创建时间，文件的读写权限列表

  • ls -S 以文件的大小进行排序列出

  • ls -d 只显示当前文件夹

  • ls -s 在每个文件的后面打印出文件的大小(size)

  • 计算当前目录下的文件数量

ll 显示所有文件

  • 基本等同于ls -al, 输出包括隐藏文件的详细信息

cd 更改目录

  • 通过~直接返回家目录
  • 跳转至家目录
  • 返回上两级目录

lsblk 列出块设备

  • 树状输出格式显示块设备,可用于查看插入usb名称
  • lsblk -l命令以列表格式显示块设备(而不是树状格式)

man 查看帮助手册

  • 查看命令的使用方法
  • 安装中文man手册并添加库函数手册

clear 清屏

ctrl+L 也可清除终端信息，ctrl+u 删除光标前，ctrl+k 删除光标后

文件操作

mkdir 创建文件夹

  • -m 选项可以指定创建目录的权限
  • -p 创建多级目录

ln 创建链接

  • -s创建软连接，-l创建硬链接
  • 软链接可以对一个不存在的文件名进行链接，而硬链接不允许
  • 软链接可以对目录进行链接，而硬链接不可以
  • 软链接可以跨文件系统 ，硬链接不可以

touch 创建文件&更改文件日期

  • -t 修改文件时间，格式为[YYMMDDhhmm]，如将日期改为2007/09/15 2:02
  • -d修改日期,如将文件日期调整为两天前
  • 创建文件

cp 复制文件/文件夹

  • -r 复制文件夹, 结尾/ 非必须，仅表示为文件夹
  • 可同时复制多个文件到一个文件夹，注意最后一个为目的文件夹
  • -u 只有源文件较目标文件新时复制, 更新文件使用，不会出错
  • -s 创建文件的软链接
  • -l 创建文件的硬链接，软连接文件和被链接文件拥有的inode不相同，硬链接文件和被链接的文件拥有相同的inode。当被链接的文件删除时，软链接文件无法访问，而硬链接文件仍能正常访问。

mv 移动文件

  • 视mv命令中第二个参数类型的不同（目标文件or目标目录），mv命令将文件重命名或将其移至一个新的目录中。当第二个参数类型是文件时，mv命令完成文件重命名，此时，源文件只能有一个，它将所给的源文件或目录重命名为给定的目标文件名。当第二个参数是已存在的目录名称时，源文件或目录参数可以有多个，mv命令将各参数指定的源文件均移至目标目录中。在跨文件系统移动文件时，mv先拷贝，再将原有文件删除，而链至该文件的链接也将丢失。
  • -b backup 若需覆盖文件，则覆盖前先行备份
  • -f force，如果目标文件已经存在，不会询问而直接覆盖
  • -i 若目标文件已经存在,询问是否覆盖
  • -u 更新(update)

rm 移除文件

  • -r &-R, 递归的删除目录下面文件以及子目录下文件

  • -v, 显示运行时详细信息

Linux 文件权限&文件时间

文件时间

  • mtime (modification time) 当文件“内容数据”（只文件里面的内容，而不是属性或权限）更改时更新, 为ls默认显示时间
  • ctime(status time) 当文件的“状态”（权限或者属性）改变时更新
  • atime (access time) 当“该文件内容被取用”（如使用cat读取）时更新
  • ls默认情况下显示mtime

文件权限

  • 开头： - 为文件，d为文件夹，l 为链接，b为USB，c为一次性内存
  • 3个一组，分别表示所有者，同组成员，其他成员所拥有的权限
  • r: (4)可读，w: (2)可写， x: (1)可执行，- :(0)无权限, 777表示所有的最大权限(4+1+2=7)

chmod 更改权限

  • chmod [who] [+ | - | =] [mode] 文件名

  • who:

    u , user，文件或目录的所有者。
    g , group, 同组用户，即与user有相同组ID的所有用户。
    o , others,其他用户
    a , all 所有用户”，包括u、g、o，为默认值

  • +为增加权限，- 取消权限，= 重分配权限

  • 如下给其他用户增加读权限，取消组用户的写权限

用户操作

top 实时查看用户进程

PID — 进程id
USER — 进程所有者
PR — 进程优先级
NI — nice值。负值表示高优先级，正值表示低优先级
VIRT — 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
RES — 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
SHR — 共享内存大小，单位kb
S — 进程状态。D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
%CPU — 上次更新到现在的CPU时间占用百分比
%MEM — 进程使用的物理内存百分比
TIME+ — 进程使用的CPU时间总计，单位1/100秒
COMMAND — 进程名称（命令名/命令行）

  • -u指定用户名，-d设置刷新延迟，-p指定查看进程，-n打印几次

  • -M按内存排序显示，-P CPU使用率排序，- I 多cpu交替显示

  • 实际内存=RES - SHR

  • & 表示任务在后台执行，如要在后台运行

运行级别(init 进程)

  • 在/etc/inittab 中可修改，只有root级能改，init进程号为1
  • 可通过如下方式查看运行级别4系统会运行程序
  • Linux 系统有 7 种运行级别(runlevel)：常用的是级别 3 和 5

运行级别 0：系统停机状态，系统默认运行级别不能设为 0，否则不能正常启动
运行级别 1：单用户工作状态，root 权限，用于系统维护，禁止远程登陆
运行级别 2：多用户状态(无 Network File System)，不支持网络
运行级别 3：完全的多用户状态(有 NFS)，登陆后进入控制台命令行模式
运行级别 4：系统未使用，保留
运行级别 5：X11 控制台，登陆后进入图形 GUI 模式
运行级别 6：系统正常关闭并重启，默认运行级别不能设为 6，否则不能正常启动
