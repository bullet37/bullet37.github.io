变量作用域&存储期限

  • static 静态变量永久保存,初始值为0，作用域取决于声明的程序块，只在程序块内可见(内部链接或无链接)
  • static只初始化一次,静态数组自动置0
  • 在程序块外部声明的变量(程序最外部)拥有文件作用域，外部链接，可被其他文件通过声明extern 变量共享
  • register变量不可通过&取得地址，不可通过指针修改，其余类似auto
  • static function 仅在定义文件中可见，更易维护，extern function 无任何特殊作用

屯屯屯屯 烫烫烫烫

  • 屯屯屯屯 为未定义堆内存，0xCDCDCD

  • 堆内存：(heap)

    程序员自己申请、自己释放，可能发生内存泄露。如new申请的堆内存。

    堆内存常用链表的方式不连续存在，大小受限于计算机系统中有效的虚拟内存，所以堆获得的空间比较灵活，也比较大。

    分配较慢，一般在堆的头部用一个字节存放堆的大小。

  • 烫烫烫烫 为未定义栈内存，0xCCCCCC

  • 栈内存：(stack)

    编译器自动分配和释放，如函数参数、局部变量、临时变量等等。

    是一块连续的内存的区域。栈顶的地址和栈的最大容量是系统预先规定好的。

  • 静态存储区：

    在程序编译的时候就已分配好，在程序的整个运行期间都存在。存放静态数据、全局数据和常量。

  • C语言中区分初始化和未初始化的，C++中不再区分了。
